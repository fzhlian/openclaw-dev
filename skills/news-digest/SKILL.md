# news-digest

按以下流程执行：

## 输入参数规范

先明确输入参数，缺失时先补问。

- 关键词：必填，支持 1 个或多个；建议优先使用名词短语
- 网站范围：必填，支持单站或多站；统一写域名，如 `openai.com`
- 时间范围：可选，默认最近 7 天
- 返回条数：可选，默认 5，最大 20
- 输出语言：可选，默认中文
- 输出模式：可选，默认“摘要总览 + 逐条清单”

当用户输入不完整时，按以下顺序补问：

1. 缺关键词时，先问追踪主题
2. 缺网站时，先问来源范围
3. 缺时间范围时，使用默认值并显式告知

## 执行流程

1. 确认参数
   - 将关键词拆分为可检索短语
   - 将网站输入统一转成域名列表

2. 检索候选内容
   - 优先组合检索词：`site:<域名> <关键词>`
   - 关键词多项时，按主题分批检索再合并去重
   - 可调用 `scripts/build_query.py` 生成查询列表

3. 提取与筛选
   - 抽取标题、发布时间、URL、核心观点
   - 去重：按 URL 和标题近似度去重
   - 过滤：广告页、导航页、弱相关内容

4. 输出结果
   - 先给“摘要总览”（3-6 条）
   - 再给“逐条摘要清单”
   - 每条包含标题、1-3 句摘要、来源站点、发布时间、链接

## 失败降级策略

- 单个站点不可访问：标注“来源受限”，继续处理其他站点
- 某关键词无结果：保留该关键词，并标注“未检索到高相关结果”
- 发布时间缺失：标注“时间未标注”，不自行推断
- 检索结果过少：降低限定条件，优先放宽时间范围，再减少站点限制
- 仍无法形成有效摘要：只输出“检索参数 + 已发现结果 + 局限说明”，不编造结论

## 质量要求

- 不编造未检索到的信息
- 对结论和来源保持一一对应
- 对时间、来源不明的信息明确标注

## 输出模板

使用以下结构：

## 摘要总览
- 主题趋势 1
- 主题趋势 2
- 主题趋势 3

## 文章清单
